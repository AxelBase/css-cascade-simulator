The Element Selector

The Hidden Power of the Element Selector

Published: December 3, 2025

Many developers treat element selectors like div, p, or section as weak and unimportant. While they are indeed the lowest-weight selector type, they are far from useless. In fact, they often become the deciding factor in the cascade when all other parts of specificity are equal.

When the Element Wins

The element selector contributes to the third position in the specificity score. When two competing rules have the same number of IDs and the same number of classes, the one that includes an element selector wins. This is true even if the other rule appears later in the stylesheet.

For example, a rule targeting header with a class will lose to div with header with the same class, because the presence of div adds one point to the final position. This subtle difference can be the reason a style unexpectedly applies or fails to apply.

Real-World Scenarios

In large projects, it is common to see rules like .card-title and h2.card-title competing. If both have one class selector, the version with the element selector wins every time. This behavior catches many developers off guard because they assume source order should dominate.

Strategic Advantage

Smart developers use this knowledge intentionally. Adding an element selector is a lightweight way to increase specificity without introducing classes or IDs. It keeps the selector readable while giving it just enough weight to win in tight situations.

Never underestimate the quiet strength of the element selector.
