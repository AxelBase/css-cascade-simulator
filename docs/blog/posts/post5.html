Pseudo-classes and Attributes in Specificity

Published: December 3, 2025

Pseudo-classes and attribute selectors are powerful tools in CSS, but many developers are unsure how they affect specificity. The answer is simple and important: they count exactly the same as class selectors in the middle position of the specificity score.

Equal Weight with Classes

A rule using hover has the same weight as one using .active. Similarly, an attribute selector like input with type equal to text counts the same as a class. This means a single pseudo-class or attribute can override a regular class selector if everything else is equal.

Common Examples

A link with hover state will beat a link with .highlight class if both rules have no other selectors. The same applies when comparing button with disabled attribute versus button with .disabled class. The browser treats them identically in terms of priority.

Practical Implications

This behavior allows clean, semantic styling. You can style form states using actual attributes rather than adding extra classes, without sacrificing control over the cascade. It also means pseudo-classes like focus-visible carry real weight and should be used intentionally.

Universal Selector Behavior

The universal selector has zero specificity. It never wins against any real selector, even an element selector. This makes it safe for global resets but useless for targeted styling.

Pseudo-classes and attributes are full citizens of the specificity system.
