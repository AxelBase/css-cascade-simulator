Compound Selectors: When Element + ID + Class Wins Everything

Published: December 3, 2025

A compound selector combines multiple selector types in one rule, such as div with header and .active. When these different types are used together, they create a specificity score that is extremely difficult to override without using equally complex selectors.

The Power of Combination

Each part of a compound selector contributes to the final score. An element selector adds to the third number, a class adds to the second, and an ID adds to the first. Even one ID in a compound selector gives it massive weight because the first number dominates all comparisons.

For example, a rule targeting div with header and .active has one ID, two classes, and one element. This creates a specificity far higher than any rule using only classes or elements, no matter how many there are.

Real-World Impact

In practice, this means a single carefully written compound selector can override entire utility frameworks or component libraries. This is both powerful and dangerous. It allows precise targeting but can make styles hard to override later.

Strategic Use

Developers use compound selectors when they need absolute confidence that a style will apply. They are common in critical UI components like headers, navigation, or modals where visual consistency is essential.

However, overusing them leads to specificity wars where increasingly complex selectors are needed to override previous ones. The best practice is to use them intentionally and sparingly.
